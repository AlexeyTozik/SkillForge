|command|description|
| --- | --- |
| <b>git init [directory]</b> | Создание пустого git-каталога в  [directory] |
| <b>git clone [repo]</b> | Клонирование удалённого git-репозитория по указанной ссылке [repo] |
| <b>git config user.name [name]</b> | Определение имени автора [name], которое будет использоваться в коммитах. Обычно используется —global флаг  |
| <b>git add <directory></b><br><br>git add src/<br>git add —all<br>git add —update | Добавление файлов/директорий [directory], которые будут отслеживаться при изменениях  (можно отдельный файл вместо директории) |
| <b>git commit -m "[message]”</b> | Фиксация коммита с именем [message] |
| <b>git status</b> | Отображает состояние рабочего каталога и раздела проиндексированных файлов.<br>С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git. |
| <b>git log</b> | Показывает историю коммитов  |
| <b>git diff [data]</b><br><br>git diff abc123 def456<br>git diff index.html<br>git diff feature-branch master | Используется для просмотра изменений между коммитами, рабочими каталогами или ветками [data] |
| <b>git branch [branch]</b><br><br>git branch<br>git branch feature-branch | Используется для создания веток в Git и просмотра всех (без аргументов) |
| <b>git chechout [branch]</b><br><br>git checkout main<br>git checkout -b feature-branch | Используется для переключения между ветками, создания новых (и переключение на них) |
| <b>git pull [remote]</b><br><branch><br>git pull origin main | Получение изменений из удаленного репозитория и автоматического объединения их с текущей веткой<br>Можно заменить на git fetch + git merge |
| <b>git push [remote] [branch]</b><br><br>git push origin main <br><i># Отправка новой ветки на удалённый репозиторий:</i><br>git push -u origin feature-branch | Используется для отправки изменений из локального репозитория в удаленный репозиторий |
| <b>git merge [branch_name]</b><br><br>git merge --abort<br>git merge --continue | Используется для объединения изменений из  [branch_name] в текущую |
| <b>git cherry-pick [commit]</b><br><br>git cherry-pick abc123<br># <i>Из нескольких коммитов:</i><br>git cherry-pick <коммит1> <коммит2> … <br># <i>Из целой ветки:</i><br>git cherry-pick feature-branch | Используется для применения изменений из одного коммита в другую ветку. Это полезно, когда вы хотите взять только определенный коммит из одной ветки и применить его к другой. |
| <b>git reset [file]</b><br><br># <i>Отмена индексации файла (git add):</i><br>git reset index.html<br># <i>Отмена последнего коммита и сохранение изменений:</i><br>git reset —soft HEAD^<br># <i>Отмена последнего коммита и удаление изменений:</i><br>git reset —hard HEAD^<br># <i>Возврат к определённому коммиту и сохранение изменений:</i><br>git reset —soft abc123<br># <i>Возврат к определённому коммиту и удаление изменений:</i><br>git reset —hard abc123 |Используется для отмены изменений |
| <b>git revert</b><br><br># <i>Создание нового коммита, который отменяет изменения указанного:</i><br>git revert abc123<br># <i>Создание серии коммитов, каждый из которых отменяет изменения в соответствующем коммите:</i><br>git revert abc123 def456<br># <i>Создание новые коммиты, отменяющие изменения в коммитах в указанном интервале:</i><br>git revert abc123^..def456 | Используется для создания нового коммита, который отменяет изменения, внесенные предыдущими коммитами.<br>В отличие от git reset, который изменяет историю коммитов, git revert создает новый коммит, который отменяет эффекты предыдущих коммитов |
| <b>git rebase [branch]</b><br><br># <i>Взять все коммиты текущей ветки, которых нет в указанной ветке, и применяет их поверх указанной ветки:</i><br>git rebase main<br># <i>Перебазирование текущей ветки с указанного коммита:</i><br>git rebase abc123<br># <i>Прервать перебазирование:</i><br>git rebase --abort | Изменение истории коммитов в Git путем перемещения, объединения или изменения коммитов |
| <b>git stash</b><br><br># <i>Временно сохраняет все изменения в стэше и возвращает рабочую директорию к состоянию последнего коммита</i><br>git stash<br># <i>Вы можете добавить комментарий, чтобы лучше понять, какие изменения были сохранены.</i><br>git stash save "Изменения для переключения на другую ветку”<br># <i>Выводит список всех сохраненных стешей</i><br>git stash list<br># <i>Применяет последний стеш, но не удаляет его из стека стешей</i><br>git stash apply<br># <i>Применяет конкретный стеш по его индексу</i><br>git stash apply stash@{2}<br># <i>Применяет последний стеш и удаляет его из стека</i><br>git stash pop<br># <i>Удаляет конкретный стеш по его индексу</i><br>git stash drop stash@{1}<br># <i>Создает новую ветку и применяет последний стеш к ней</i><br>git stash branch new_branch_name| Используется для временного сохранения изменений, которые вы внесли в рабочую директорию, но вы не хотите коммитить прямо сейчас. Это может быть полезно, когда вы хотите переключиться на другую ветку, но у вас есть незакоммиченные изменения. |
| <b>git rm [file]</b><br><br><brgit rm index.html<br># <i>Удаление файла из индекса:</i><br>git rm --cached index.html<br># <i>Удаление файла и создание коммита:</i><br>git rm -m “FIle ABC.txt is deleted”<br>git rm '*.txt’ | Используется для удаления файлов из рабочей директории и индекса Git |
